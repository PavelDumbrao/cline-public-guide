# README FOR CLINE

> Это инструкция для AI-агента, который читает этот публичный репозиторий как шаблон настройки Cline.

Ты работаешь внутри публичного учебного репозитория по Cline.

## Цель
Помогать пользователю настраивать и адаптировать Cline безопасно, без утечки секретов и без слепого копирования чужой инфраструктуры.

## Что важно
- Всегда различай rules, skills, hooks, MCP и workflows.
- Не путай Cline Workflows и n8n workflow JSON.
- Не предлагай копировать личный tech stack автора как обязательный стандарт.
- Не храни секреты в markdown, skill-файлах, rules и hooks.
- Если встречаешь placeholders — ожидай, что пользователь должен заменить их вручную.

## Как использовать этот репозиторий
1. Сначала читай `ai/rules/`.
2. Потом при необходимости подключай `ai/skills/`.
3. Если задача повторяющаяся — используй `ai/workflows/`.
4. Если нужна автоматизация событий — смотри `ai/hooks/`.
5. Для интеграции с n8n используй примеры из `examples/n8n/` только как шаблоны.

## Если пользователь хочет собрать свой setup по этому repo
- Сначала сделай аудит его задач и текущего проекта.
- Потом предложи **минимальный** набор rules / skills / MCP.
- Не копируй весь репозиторий механически.
- Всё адаптируй под задачи пользователя и его стек.
- После каждого этапа проверяй результат, а не просто объявляй «готово».
- Если нужны пользовательские значения — оставляй placeholders и явно говори, что заменить вручную.

## Главные принципы
- Практичность важнее красоты.
- Проверка результата обязательна.
- Минимальная рабочая конфигурация лучше перегруженной.
- Всё личное должно быть заменено шаблонами.
