# Smoke Test

> Этот skill нужен для простой, быстрой проверки: **после изменений система вообще работает или уже сломана на базовом уровне**.

Сначала полезно прочитать:
- [`../deployment-guide/SKILL.md`](../deployment-guide/SKILL.md) — если smoke test идёт после выката
- [`../../rules/07-tool-usage.md`](../../rules/07-tool-usage.md) — чтобы не объявлять задачу завершённой без реальной проверки
- [`../../rules/10-browser-automation-safe.md`](../../rules/10-browser-automation-safe.md) — если smoke test включает браузерную проверку

---

## Что это за skill простыми словами

Smoke test — это не полный тест-план.
Это быстрый ответ на вопрос:

**«После изменений всё ещё живо на базовом уровне или уже нет?»**

Он нужен, чтобы не говорить «готово», пока не проверен хотя бы основной пользовательский сценарий.

---

## Когда использовать
- После деплоя
- После заметных изменений
- Перед объявлением задачи завершённой

## Чек-лист
- Сервис открывается
- Основной сценарий работает
- Нет критических ошибок в консоли/логах
- Нет падения на первом действии пользователя

## Типичные ошибки
- Завершать задачу без проверки
- Проверять только один слой и забывать про остальные

## Как проверить
Сделай 1–3 ключевых пользовательских сценария и убедись, что они проходят.
