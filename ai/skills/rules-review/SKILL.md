# Rules Review

## Когда использовать
- Перед публикацией rule-набора
- Когда rules разрослись
- Когда агент ведёт себя нестабильно

## Что проверять
- Повторы
- Личные данные
- Смешение rules и skills
- Слишком длинные и расплывчатые инструкции

## Типичные ошибки
- Incident notes внутри rules
- Секреты в markdown
- Rules, которые описывают частный случай как общий стандарт

## Как проверить
Для каждого rule ответь:
- reusable?
- safe?
- понятен новичку?
- не содержит личной инфры?
