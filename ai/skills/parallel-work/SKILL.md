# Parallel Work

> Этот skill нужен, когда задачу выгодно исследовать в несколько независимых потоков, а не идти строго одной линией.

Сначала полезно прочитать:
- [`../../rules/03-context-and-memory.md`](../../rules/03-context-and-memory.md) — почему большие задачи надо дробить
- [`../../rules/07-tool-usage.md`](../../rules/07-tool-usage.md) — чтобы параллельность не превратилась в хаос

---

## Что это за skill простыми словами

Параллельная работа — это не «делать всё сразу».
Это значит:
- разделить независимые подзадачи
- заранее договориться, кто что исследует
- потом собрать результат в один понятный вывод

Если не определить контракт — получится не ускорение, а бардак.

---

## Когда использовать
- Нужно исследовать несколько файлов или направлений сразу
- Есть независимые подзадачи
- Нужен best-of-N для сложного решения

## Правила
- Сначала определи контракты
- Разделяй независимые задачи
- После параллельной работы делай единый review
- Не путай параллельность с хаосом

## Типичные ошибки
- Параллелить зависимые шаги
- Не договориться о формате результата
- Не делать финальную сверку

## Как проверить
- Есть ли понятный итог от каждого потока
- Нет ли конфликтов в решениях
- Можно ли собрать общий результат без догадок

---

## ✅ Чек-лист для самопроверки
- [ ] Я параллелю только независимые части задачи
- [ ] Для каждого потока заранее понятен результат на выходе
- [ ] После параллельной работы есть единая сверка
- [ ] Параллельность уменьшила хаос, а не усилила его
