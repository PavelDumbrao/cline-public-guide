# Connect Automation

## Когда использовать
- Нужно соединить несколько сервисов
- Нужно спроектировать webhook/API-процесс
- Нужно автоматизировать повторяющееся действие

## Подход
1. Определи вход
2. Определи обработку
3. Определи выход
4. Добавь обработку ошибок
5. Проверь повторяемость и идемпотентность

## Типичные ошибки
- Нет явного входа/выхода
- Нет retry/error path
- Нет понимания, где хранится состояние

## Как проверить
- Входной event реально проходит весь путь
- Ошибки не теряются
- Повторный запуск не ломает процесс
