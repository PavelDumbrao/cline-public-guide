# n8n HTTP Request

> Этот skill нужен, когда в n8n надо не просто «сделать запрос», а аккуратно подключить внешний API так, чтобы потом не искать ошибку вслепую.

- [`../connect-automation/SKILL.md`](../connect-automation/SKILL.md) — как мыслить об автоматизации целиком, а не только об одном HTTP-вызове

---

## Что это за skill простыми словами

HTTP Request в n8n — это точка, где твой workflow общается с внешним миром.

Здесь чаще всего ломается всё из-за мелочей:
- не тот метод
- не тот header
- не тот body
- не тот формат ответа

Этот skill нужен, чтобы не стрелять по API наугад, а собирать запрос осознанно.

---

## Когда использовать
- Нужно вызвать внешний API из n8n
- Нужно отправить POST/GET/PATCH/DELETE
- Нужно настроить headers, auth или body

## Что проверять
- Метод
- URL
- Заголовки
- Тело запроса
- Формат ответа
- Обработка ошибок

## Типичные ошибки
- Неправильный content-type
- Неверный формат body
- Потерянные headers авторизации
- Отсутствие проверки status code

## Как проверить
- Запрос возвращает ожидаемый статус
- Ответ парсится корректно
- Ошибки понятны и обрабатываются
