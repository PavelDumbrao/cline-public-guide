# Deployment Guide

> Этот skill нужен, чтобы деплой не выглядел как «нажал кнопку и надеюсь, что заработает».

Сначала полезно прочитать:
- [`../smoke-test/SKILL.md`](../smoke-test/SKILL.md) — как проверять систему после выката
- [`../../rules/06-code-quality.md`](../../rules/06-code-quality.md) — чтобы не везти в продакшен хрупкие правки

---

## Что это за skill простыми словами

Этот skill помогает пройти путь от кода до рабочего сервиса без лишней паники.

Его задача — напомнить, что деплой состоит не только из «выложить», но и из:
- выбора платформы
- проверки env
- проверки команд запуска
- проверки после выката

---

## Когда использовать
- Нужно выбрать платформу
- Нужно подготовить проект к деплою
- Нужно описать безопасный процесс выката

## Минимальный процесс
1. Определи тип проекта
2. Выбери платформу
3. Подготовь env-переменные
4. Проверь build/start команды
5. Выкати
6. Проверь health и логи

## Типичные ошибки
- Деплой без проверки env
- Отсутствие smoke test после выката
- Смешивание локальных и продовых настроек

## Как проверить
- Сервис стартует
- Основная страница/endpoint отвечает
- Нет критических ошибок в логах
