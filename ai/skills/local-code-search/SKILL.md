# Local Code Search

> Этот skill нужен, когда ответ, скорее всего, уже **есть внутри проекта**, но ты ещё не знаешь точное имя файла, функции или класса.

Сначала полезно прочитать:
- [`../../rules/05-search-tools.md`](../../rules/05-search-tools.md) — чтобы понимать, когда искать в проекте, а когда идти в docs или веб
- [`../../docs/07-mcp-guide-human.md`](../../docs/07-mcp-guide-human.md) — если используешь `cocoindex-code` или похожий MCP для смыслового поиска

---

## Что это за skill простыми словами

Этот skill нужен, чтобы не читать проект целиком вручную.

Идея простая:
- сначала ищешь **по смыслу**
- потом быстро читаешь лучшие найденные места
- потом уже решаешь, где менять код

Это особенно полезно в больших репозиториях, где обычный поиск по точному слову часто не помогает.

---

## Когда использовать
- Репозиторий большой
- Не знаешь точное имя файла/функции
- Нужно быстро найти место реализации

## Подход
1. Сначала ищи смыслом
2. Потом читай 1–3 лучших результата
3. Потом уточняй запрос

## Типичные ошибки
- Читать весь проект подряд
- Искать только точными словами
- Дублировать существующую логику, не найдя её

## Как проверить
- Найден релевантный участок кода
- Понятно, почему он релевантен
- Дальше можно принимать решение по изменению
