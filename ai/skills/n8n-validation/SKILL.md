# n8n Validation

## Когда использовать
- Перед публикацией workflow
- После крупных изменений
- Если workflow не запускается как ожидалось

## Что проверять
- обязательные поля нод
- выражения
- связи между узлами
- credentials
- error path

## Типичные ошибки
- Пустые обязательные параметры
- Ссылки на несуществующие данные
- Захардкоженные секреты
- Непонятная маршрутизация

## Как проверить
- workflow валиден
- ключевые ветки выполняются
- чувствительные данные вынесены наружу
