# n8n Validation

> Этот skill нужен, чтобы workflow выглядел не просто «собранным», а реально проверенным перед запуском или публикацией.

Сначала полезно прочитать:
- [`../n8n-workflow-patterns/SKILL.md`](../n8n-workflow-patterns/SKILL.md) — как строить workflow так, чтобы его вообще можно было валидировать и поддерживать
- [`../n8n-http-request/SKILL.md`](../n8n-http-request/SKILL.md) — если внутри workflow есть внешние API-вызовы

---

## Что это за skill простыми словами

Этот skill нужен, когда ты хочешь ответить на вопрос:

**«Workflow просто выглядит собранным или действительно готов к работе?»**

Он помогает проверить:
- обязательные поля
- выражения
- связи между нодами
- credentials
- error path

---

## Когда использовать
- Перед публикацией workflow
- После крупных изменений
- Если workflow не запускается как ожидалось

## Что проверять
- обязательные поля нод
- выражения
- связи между узлами
- credentials
- error path

## Типичные ошибки
- Пустые обязательные параметры
- Ссылки на несуществующие данные
- Захардкоженные секреты
- Непонятная маршрутизация

## Как проверить
- workflow валиден
- ключевые ветки выполняются
- чувствительные данные вынесены наружу

---

## ✅ Чек-лист для самопроверки
- [ ] В workflow нет пустых обязательных полей
- [ ] Выражения ссылаются на реальные данные
- [ ] Ветки и связи понятны глазами
- [ ] Секреты не захардкожены внутри workflow
