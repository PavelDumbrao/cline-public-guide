# Frontend Design

> Этот skill нужен, когда задача не просто «сверстать что-то», а сделать интерфейс, который реально понятен человеку и нормально живёт в проекте.

- [`../smoke-test/SKILL.md`](../smoke-test/SKILL.md) — как делать быструю проверку после изменений

## Что это за skill простыми словами

Этот skill помогает не прыгать сразу в JSX/CSS, а сначала понять:
- что должен делать экран
- какой сценарий проходит пользователь
- что будет в пустом, ошибочном и загрузочном состоянии

То есть сначала логика UX, потом компоненты.

---

## Когда использовать
- Нужно собрать интерфейс с нуля
- Нужно улучшить UX существующей страницы
- Нужно сделать адаптивный и понятный UI

## Когда НЕ использовать
- Задача не про интерфейс
- Пользователь просит только backend или инфраструктуру

## Принципы
- Сначала сценарий пользователя, потом компоненты
- Простота важнее визуальной перегрузки
- Дефолтно учитывай мобильный экран
- Делай понятные тексты и состояния загрузки/ошибок

## Типичные ошибки
- Сразу писать компоненты без структуры экранов
- Игнорировать пустые состояния и ошибки
- Перегружать интерфейс декоративными элементами

## Как проверить
- Открой страницу в браузере
- Проверь mobile/desktop
- Проверь пустые и ошибочные состояния
