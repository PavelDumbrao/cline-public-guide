# Supabase Integration

> Этот skill нужен, когда ты подключаешь Supabase и важно не просто заставить запросы работать, а не допустить утечки ключей и ошибок в правах доступа.

Сначала полезно прочитать:
- [`../../rules/06-code-quality.md`](../../rules/06-code-quality.md) — про секреты, ошибки и безопасные изменения
- [`../../rules/02-documentation-search.md`](../../rules/02-documentation-search.md) — если надо сверять актуальный синтаксис клиента или auth

---

## Что это за skill простыми словами

Supabase часто кажется простым, пока не начинаются ошибки с:
- ключами
- auth
- RLS
- правами на чтение/запись

Этот skill нужен, чтобы сразу думать безопасно:
- что можно держать в клиенте
- что нельзя выносить в браузер
- какие права реально получит пользователь

---

## Когда использовать
- Нужно подключить Supabase к frontend/backend
- Нужно настроить auth или работу с таблицами

## Базовые правила
- URL и ключи только из env
- Разделяй anon key и service key
- Проверяй RLS
- Не клади секреты в клиентский код

## Типичные ошибки
- Использование service key в браузере
- Отсутствие RLS
- Неправильные env имена

## Как проверить
- Клиент подключается
- Запросы проходят с ожидаемыми правами
- Нет утечки секретных ключей
