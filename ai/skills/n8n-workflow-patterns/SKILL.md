# n8n Workflow Patterns

> Этот skill нужен, когда важно не просто «накидать ноды», а собрать workflow с понятной архитектурой и читаемым путём данных.

Сначала полезно прочитать:
- [`../n8n-validation/SKILL.md`](../n8n-validation/SKILL.md) — чтобы потом проверять workflow не только визуально, но и по структуре
- [`../connect-automation/SKILL.md`](../connect-automation/SKILL.md) — чтобы видеть автоматизацию как вход → обработка → выход

---

## Что это за skill простыми словами

Этот skill нужен, чтобы workflow был понятен не только его автору, но и другому человеку через неделю.

Хороший паттерн помогает:
- не смешивать всё в одну большую ноду
- явно показать, где вход, где валидация, где обработка, где ответ
- заранее предусмотреть error path

---

## Когда использовать
- Нужно выбрать структуру workflow
- Нужно разложить большой сценарий на понятные узлы

## Базовые паттерны
- Webhook → Normalize → Validate → Process → Respond
- Trigger → Fetch → Transform → Store → Notify
- Router → Branches → Join / finish

## Типичные ошибки
- Слишком много логики в одном node
- Нет нормализации входа
- Нет явного error path

## Как проверить
- По workflow можно глазами понять путь данных
- Ветки логичны
- Есть обработка невалидных входов
