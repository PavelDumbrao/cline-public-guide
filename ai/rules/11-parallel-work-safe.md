# ⚡ Parallel Work

> Это правило про безопасную параллельность: как ускорять исследование задачи, не превращая её в неуправляемый шум.

Сначала полезно прочитать:
- [`03-context-and-memory.md`](03-context-and-memory.md) — почему большие задачи нужно дробить
- [`07-tool-usage.md`](07-tool-usage.md) — как не терять контроль над процессом

## Когда использовать параллельность
- Нужно быстро исследовать несколько частей проекта
- Есть независимые подзадачи
- Нужен best-of-N подход

## Правила
- Сначала определи контракты
- Не смешивай зависимые задачи в один параллельный запуск
- После параллельной работы делай единый review результата
