# Parallel Work

## Когда использовать параллельность
- Нужно быстро исследовать несколько частей проекта
- Есть независимые подзадачи
- Нужен best-of-N подход

## Правила
- Сначала определи контракты
- Не смешивай зависимые задачи в один параллельный запуск
- После параллельной работы делай единый review результата
