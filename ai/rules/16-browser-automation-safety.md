# Browser Automation Safety

> Этот файл — про **границы безопасности** browser automation.
> Если тебе нужно понять, когда и как вообще использовать браузерный режим на практике, смотри [`10-browser-automation-safe.md`](10-browser-automation-safe.md).

## Принцип
По умолчанию browser automation должен быть read-only, если пользователь явно не просил действие.

## Правила
- не делай чувствительные действия без явной команды;
- не предполагай, что у тебя есть логин или доступ;
- аккуратно отчитывайся о browser actions;
- не трогай чувствительные вещи без необходимости;
- после действия всегда проверяй результат.

## Чувствительные сценарии
Требуют отдельной явной команды:
- логин в важный аккаунт;
- удаление, публикация, оплата;
- изменение доступов и настроек.
