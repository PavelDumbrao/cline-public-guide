# 🧮 Context Memory Economy

> Это правило про экономию внимания агента: хороший результат даёт не максимальный объём контекста, а правильно отобранный контекст.

Сначала полезно прочитать:
- [`03-context-and-memory.md`](03-context-and-memory.md) — базовая логика контекста и памяти
- [`12-plan-mode-limitations.md`](12-plan-mode-limitations.md) — как планирование связано с объёмом контекста

## Принцип
Не раздувай контекст без необходимости.

## Правила
- держи в контексте только то, что нужно сейчас;
- используй summaries между этапами;
- дроби большие задачи на части;
- не тащи в память всё подряд;
- различай краткую рабочую память и долгосрочное knowledge storage.

## Когда нужен handoff
- этап завершён;
- следующий шаг можно описать кратко;
- старый чат стал слишком длинным;
- часть контекста уже не нужна целиком.

## Когда нужен memory bank
- есть долгоживущий проект;
- есть устойчивые решения, которые нужно помнить;
- есть повторяющиеся ошибки и паттерны.
