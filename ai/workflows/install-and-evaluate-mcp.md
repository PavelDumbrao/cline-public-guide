# /install-and-evaluate-mcp

## Цель
Понять, нужен ли новый MCP, подключить его и проверить, приносит ли он реальную пользу setup.

## Когда использовать
- появился новый use-case;
- текущего MCP-набора уже не хватает;
- нужно понять, не дублирует ли новый MCP существующий.

## Входные данные
- имя MCP;
- use-case;
- понимание, нужен ли ему API key / аккаунт.

## Пошаговый процесс
1. Понять задачу, которую должен решать MCP
2. Проверить, не закрывает ли её уже существующий MCP
3. Подключить MCP
4. Кратко объяснить, зачем он нужен
5. Прогнать минимальный smoke test
6. Решить: оставить, отложить или убрать

## Что считать хорошим результатом
- MCP реально решает понятную задачу
- не дублирует существующий стек
- его назначение понятно пользователю

## Что проверить после выполнения
- MCP реально вызывается
- польза доказана хотя бы одним кейсом
- нет лишней сложности в setup

## Типичные ошибки
- ставить MCP “на всякий случай”
- не понимать его роль
- не делать smoke test

## Ограничения по безопасности
- не хранить секреты в repo
- не подключать private-only MCP без реальной причины

## Expected output
- решение, нужен MCP или нет
- краткое объяснение роли MCP
- smoke test
- рекомендация по месту в общем стеке

## Minimal report format
- что найдено / сделано
- что изменено
- что требует ручной проверки
- где риски
- следующий шаг
