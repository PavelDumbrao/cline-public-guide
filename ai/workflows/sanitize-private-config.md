# /sanitize-private-config

## Цель
Очистить рабочие артефакты от приватных данных перед публикацией или передачей другим людям.

## Когда использовать
- перед публикацией repo
- перед отправкой архива или примеров
- перед созданием публичного гайда

## Входные данные
- markdown-файлы
- scripts
- workflow JSON
- env-примеры
- config-файлы

## Пошаговый процесс
1. Найди все sensitive поля
2. Отметь, что можно заменить placeholder'ом
3. Удали credentials blocks там, где они не нужны
4. Замени tokens / keys / ids / URLs / IP / domains
5. Проверь, что после очистки файл всё ещё понятен
6. Прогони финальный grep/review на утечки

## Что считать хорошим результатом
- секретов нет
- смысл файлов сохранён
- новичок понимает, что надо заменить вручную

## Что проверить после выполнения
- нет ли raw tokens
- нет ли credentials
- нет ли личных доменов/IP
- нет ли user-specific идентификаторов

## Типичные ошибки
- удалить слишком много и сделать файл бесполезным
- забыть file IDs и credential names
- оставить служебные URL или IP в комментариях

## Ограничения по безопасности
- при сомнении лучше заменить на placeholder
- если файл нельзя безопасно очистить — не публиковать
