# /create-skill-from-docs

## Цель
Сделать reusable skill из документации, а не просто сохранить note.

## Когда использовать
- тема повторяется;
- по документации часто приходится отвечать заново;
- нужен устойчивый шаблон действий.

## Входные данные
- документация / ссылки / summary
- понимание повторяющегося use-case

## Пошаговый процесс
1. Изучи docs
2. Выдели повторяющуюся тему
3. Реши, нужен ли skill, а не просто note
4. Создай структуру skill
5. Добавь:
   - when to use
   - when not to use
   - common mistakes
   - validation
6. Проверь skill на простом сценарии

## Что считать хорошим результатом
- skill reusable
- skill не слишком общий
- skill помогает отвечать стабильнее и практичнее

## Что проверить после выполнения
- skill можно вызвать явно
- skill работает на простом кейсе
- skill не содержит секретов и private-only деталей

## Типичные ошибки
- делать note вместо skill
- копировать docs без переработки
- не добавлять валидацию и ошибки

## Ограничения по безопасности
- не переносить приватные данные из docs
- всё чувствительное заменять placeholder-ами

## Expected output
- новый skill
- его назначение
- пример простого использования
- список ручных адаптаций, если они нужны

## Minimal report format
- что найдено / сделано
- что изменено
- что требует ручной проверки
- где риски
- следующий шаг
