# /recover-from-model-drift

## Цель
Восстановить рабочий ход после зацикливания, галлюцинаций или потери роли моделью.

## Когда использовать
- модель повторяется;
- модель спорит с задачей;
- модель не держит ограничения;
- ответы явно деградировали.

## Symptom checklist
- повторяет один и тот же неверный ответ
- забывает ограничения
- путает файлы и факты
- уходит в длинную воду вместо действия
- просит лишние переключения режима без пользы

## Recovery actions
1. Остановить текущий цикл
2. Зафиксировать симптом
3. Проверить, не перегружен ли контекст
4. Сократить вводные до краткой версии
5. При необходимости переключить модель
6. Если нужно — начать новую сессию
7. Если задача архитектурная — отдать reasoning внешней модели

## When to switch model
- если текущая модель дважды подряд уходит не туда
- если нужен другой тип мышления
- если контекст уже испорчен

## When to restart context/session
- если чат перегружен
- если слишком много старых веток обсуждения
- если модель не восстанавливается коротким reset

## Expected output
- диагностированный симптом drift
- выбранное действие восстановления
- короткий план продолжения работы

## Minimal report format
- что найдено / сделано
- что изменено
- что требует ручной проверки
- где риски
- следующий шаг
