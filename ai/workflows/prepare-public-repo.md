# /prepare-public-repo

## Цель
Подготовить публичный репозиторий из живой рабочей системы без утечки приватных данных.

## Когда использовать
- когда есть зрелый локальный setup
- когда нужно сделать учебный или open-source шаблон
- когда нужно передать систему ученикам или команде

## Входные данные
- текущий рабочий setup
- список rules / skills / hooks / MCP
- примеры workflows и docs

## Пошаговый процесс
1. Раздели материалы на "для человека" и "для AI"
2. Отдели Cline Workflows от n8n JSON
3. Найди и убери private-only части
4. Замени sensitive данные placeholders
5. Оставь только reusable шаблоны
6. Сделай README с маршрутами для новичка
7. Добавь docs по переносу и проверке
8. Прогони self-review перед публикацией

## Что считать хорошим результатом
- новичок понимает, куда идти
- AI получает отдельный чистый блок инструкций
- нет секретов и личной инфраструктуры
- repo можно адаптировать под другой setup

## Что проверить после выполнения
- ссылки работают
- структура понятна
- нет путаницы между сущностями
- sensitive review пройден

## Типичные ошибки
- зеркалить живую систему вместо шаблона
- не разделять human/AI слои
- оставить credentials в JSON-артефактах

## Ограничения по безопасности
- не публикуй реальные токены
- не публикуй продовые endpoints
- не тащи личный tech-stack как стандарт
