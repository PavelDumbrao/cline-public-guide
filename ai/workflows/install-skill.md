# /install-skill

## Цель
Безопасно добавить новый skill в проект или личный setup.

## Когда использовать
- когда новая тема повторяется регулярно
- когда нужна устойчивая инструкция для Cline
- когда не хочется каждый раз писать одно и то же заново

## Входные данные
- название skill
- краткое назначение
- список сценариев использования
- решение, где skill будет храниться

## Пошаговый процесс
1. Проверь, нужен ли skill вообще
2. Проверь, нет ли уже похожего skill
3. Выдели тему и границы skill
4. Создай папку `skill-name/`
5. Создай `SKILL.md`
6. Добавь:
   - когда использовать
   - когда не использовать
   - пошаговый подход
   - типичные ошибки
   - как проверить результат
7. Проверь, что внутри нет секретов и личной инфраструктуры
8. Подключи и протестируй на одной реальной задаче

## Что считать хорошим результатом
- skill понятен по названию
- его можно вызывать явно
- он улучшает ответы на своей теме
- в нём нет private-only данных

## Что проверить после выполнения
- Cline реально использует skill по назначению
- skill не дублирует другой skill
- новичок понимает, зачем он нужен

## Типичные ошибки
- делать skill слишком общим
- класть в skill секреты
- путать skill с rule
- не указывать "когда не использовать"

## Ограничения по безопасности
- никаких токенов и URL продакшена
- никаких project-specific секретов
- всё чувствительное только через placeholders
