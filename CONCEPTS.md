# CONCEPTS

Этот файл нужен, чтобы новичок быстро понял основные сущности Cline и не начал путать их между собой.

## 1. Rule
Rule — это постоянное правило поведения AI.

Примеры:
- на каком языке отвечать
- как искать документацию
- как обращаться с секретами
- как использовать инструменты

## 2. Skill
Skill — это специализированный модуль знаний.

Примеры:
- как делать системную отладку
- как работать с Supabase
- как строить n8n workflow

## 3. Hook
Hook — это bash-скрипт, который срабатывает на событии Cline.

Примеры:
- перед запуском команды
- после запуска инструмента
- при старте задачи

## 4. MCP
MCP — это подключаемый источник данных или инструмент.

Примеры:
- документация библиотек
- браузер
- веб-поиск
- GitHub

## 5. Cline Workflow
Cline Workflow — это markdown-процесс для самого Cline.
Он описывает шаги, по которым агент должен пройти.

## 6. n8n Workflow JSON
Это JSON-файл, который импортируется в n8n.
Это не часть механики Cline.

## 7. Почему это нельзя путать
Если смешать эти сущности, получится каша:
- rules станут слишком длинными
- skills будут дублировать hooks
- workflow начнут путать с JSON-артефактами
- новичок перестанет понимать систему

## 8. Практичная модель
- GPT / Perplexity = архитектор
- Cline = исполнитель
- человек = проверяющий и принимающий решение

## 9. Главный принцип
Публичный репозиторий должен быть шаблоном.
Он не должен быть копией живой приватной системы автора.
