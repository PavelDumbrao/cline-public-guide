# 17. Context, Memory and Compact

## Почему длинный контекст дорогой
Чем длиннее чат, тем:
- больше токенов сгорает;
- сложнее модели удерживать главное;
- выше шанс, что она начнёт путать старое и новое;
- больше мусора тянется в каждый следующий ответ.

## Почему “тащить всё в один чат” — плохая идея
Один длинный чат часто приводит к трём проблемам:
1. модель теряет фокус;
2. реальные факты начинают смешиваться с устаревшими;
3. сложнее понять, где именно началась ошибка.

## Когда задача уже раздулась
Сигналы:
- ты уже плохо помнишь, что было в начале;
- модель отвечает всё менее точно;
- в задаче уже много независимых подтем;
- приходится постоянно напоминать одно и то же;
- чат стал длиннее, чем сама полезная работа.

## Что такое summary / compact / handoff простыми словами
- **summary** — короткая выжимка того, что важно сохранить;
- **compact** — сокращение текущего контекста до сути;
- **handoff** — передача следующей сессии того, что уже сделано и что осталось.

## Зачем нужен memory bank
Memory bank нужен не для хранения всего подряд, а для хранения **важного и стабильного**:
- ключевых решений,
- текущего прогресса,
- повторяющихся ошибок,
- структуры проекта.

## Когда дробить задачу на куски
Дроби задачу, если:
- её можно разделить на независимые этапы;
- один этап укладывается в 15–30 минут;
- уже тяжело держать весь объём в голове;
- надо часто проверять промежуточный результат.

## Как понять, что Cline начинает тонуть в контексте
- растёт количество неточных или повторяющихся ответов;
- модель начинает забывать ограничения;
- ответы становятся слишком общими;
- приходится снова объяснять структуру задачи.

## Как сочетать разные уровни памяти

### Текущий рабочий контекст
Используй для того, что нужно прямо сейчас.

### Краткие summaries
Используй для перехода между этапами задачи.

### Долгосрочная память / knowledge storage
Используй для устойчивых фактов, а не для мусора.

### Внешний архитектор (GPT / Perplexity)
Используй, когда нужно обдумать следующий шаг без засорения рабочего чата Cline.

## Чеклист: контекст уже перегружен?
- [ ] чат стал очень длинным
- [ ] модель начала повторяться
- [ ] есть несколько независимых подтем
- [ ] ты сам уже теряешь нить
- [ ] приходится постоянно напоминать базовые вводные

Если отмечено 2–3 пункта — контекст уже пора сокращать.

## Чеклист: пора делать compact / handoff?
- [ ] этап завершён
- [ ] есть понятный список сделанного
- [ ] есть список оставшегося
- [ ] следующий шаг можно описать коротко
- [ ] нет смысла тащить весь старый чат дальше

## Практическое правило
Лучше несколько коротких управляемых сессий, чем одна гигантская и умирающая от перегруза.
