## 🤖 Что такое Cline — честно и без лишнего

Cline — это AI-агент внутри VS Code. Он не просто отвечает на вопросы — он **действует**:
читает твои файлы, изменяет код, запускает команды в терминале, открывает браузер, вызывает внешние API.

Это не ChatGPT в редакторе. Это исполнитель, которому ты даёшь задачи — и он их выполняет.

И именно поэтому важно настроить его **правильно**, а не просто включить и надеяться на лучшее.

## 🧠 Четыре вещи, которые тебе нужно понять сразу

В Cline есть четыре базовые сущности. Без понимания этих четырёх — всё остальное бессмысленно.

### Rules — правила поведения
Это инструкции, которые Cline читает **всегда**, при каждой задаче.
Они задают: на каком языке отвечать, как обрабатывать ошибки, что делать с секретами, как оценивать уверенность.

Правила — это характер твоего агента.

### Skills — специализированные навыки
Активируются только когда нужны. Например: скилл по работе с n8n загружается только тогда, когда задача касается n8n. В остальное время он не занимает контекст.

Скиллы — это экспертиза агента по конкретным темам.

### Hooks — автоматические триггеры
Баш-скрипты, которые срабатывают на события: задача началась, задача завершилась, агент собирается выполнить команду.
Можно использовать для логирования, уведомлений, защиты от опасных действий.

Хуки — это автоматика вокруг работы агента.

### MCP — внешние инструменты
MCP-серверы дают агенту доступ к внешним сервисам: поиск в интернете (Tavily), документация библиотек (Context7), браузер (Playwright), GitHub, Railway и другие.

Без MCP — агент работает только с тем, что есть у тебя локально.

## ⚠️ Главная ошибка, которую не надо повторять

Даже если тебе дали ссылку на чью-то полную настройку — у того человека:
- свои сервисы и инфраструктура

Она будет работать плохо, а ты не будешь понимать почему.

## ⏱️ Первый запуск: 15 минут до рабочей системы

### 1. Скопируй минимальный набор rules
Возьми 5–7 файлов из `ai/rules/` (список выше). Положи их в свой проект в папку `.clinerules/`.

### 2. Скопируй 3–4 skills
Возьми из `ai/skills/` и положи в `~/.agents/skills/` (глобальная папка) или в `.agents/skills/` внутри проекта.

### 3. Установи MCP
Сначала поставь Context7 и Tavily — через настройки Cline → MCP Servers. Если тебе реально нужны сайты, формы и UI-проверки, потом отдельно добавь Playwright. Инструкция: [`docs/07-mcp-guide-human.md`](07-mcp-guide-human.md)

### 4. Запусти smoke test
Проверь что всё работает: [`examples/prompts/smoke-tests.md`](../examples/prompts/smoke-tests.md)

## 🧪 Пять проверочных задач для первого запуска

Дай Cline эти пять задач по очереди. Если всё работает — ты настроил систему правильно.

**1. Проверка языка**
```text
Ответь по-русски и предложи краткий план из 3 шагов для создания README.
```

**2. Проверка поиска документации**
```text
Найди в официальной документации, как работает fetch timeout в JavaScript.
```

**3. Проверка безопасности**
```text
Что произойдёт если выполнить git add .env? Объясни без выполнения команды.
```

**4. Проверка активации скилла**
```text
Используй systematic-debugging и покажи как расследовать 500 ошибку API.
```

**5. Проверка браузера (только если ты ставил Playwright)**
```text
Открой localhost:3000 в браузере и проверь, нет ли ошибок в консоли.
```

- [ ] Агент умеет активировать skill по задаче
