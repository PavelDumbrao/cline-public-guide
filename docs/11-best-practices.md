## 1. Не копируй чужой tech stack вслепую
Чужая система может быть сильной, но она выросла под другой контекст.

## 2. Держи публичный repo шаблонным
Публичный репозиторий должен объяснять, а не тащить за собой приватную операционку автора.

## 3. Начинай с малого
Минимум rules, skills и MCP лучше, чем огромная система без понимания.

## 4. Не верь AI без проверки
Проверка результата — отдельная фаза, а не опция.

## 5. Не храни секреты в markdown
Даже если кажется, что это «временно».

## 6. Делай явное разделение
- для человека
- для AI
- примеры
- шаблоны

## 7. Думай ролями
- GPT/Perplexity = архитектор
- Cline = исполнитель
- человек = финальный контролёр
