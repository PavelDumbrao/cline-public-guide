# 21. Learning Path for Students

## Шаг 1. Понять сущности
### Что изучить
- rules
- skills
- hooks
- MCP
- workflows

### Домашка
Объясни своими словами разницу между rule, skill и hook.

### Критерий понимания
Ты можешь без подсказки сказать, что куда относится.

### Типичный провал
Ученик сваливает всё в rules.

## Шаг 2. Поставить базовый setup
### Что сделать
- взять 5–7 rules
- взять 3–4 skills
- поставить 2–3 MCP

### Домашка
Собери минимальный setup без hooks.

### Критерий понимания
Ты не копируешь всё подряд, а понимаешь, зачем каждый элемент нужен.

### Типичный провал
Сразу поставить весь стек автора.

## Шаг 3. Научиться проверять setup
### Что сделать
Прогнать smoke tests на:
- language/style
- docs search
- skills
- MCP

### Домашка
Пройти готовые prompt-тесты из [`examples/prompts/smoke-tests.md`](../examples/prompts/smoke-tests.md).

### Критерий понимания
Ты умеешь доказать, что setup работает, а не просто «кажется, что работает».

### Типичный провал
Ничего не проверять после переноса.

## Шаг 4. Освоить MCP и browser
### Что сделать
Понять, когда нужен search, browser, docs, local code search.

### Домашка
Собрать свой минимальный MCP-набор под одну конкретную роль.

### Критерий понимания
Ты можешь объяснить, почему не ставишь лишний MCP.

### Типичный провал
Плодить мусорный MCP-стек.

## Шаг 5. Сделать первый мини-проект
### Что сделать
Собери маленький рабочий кейс с rules + skills + MCP.

### Домашка
Например: мини-репозиторий с README, одним skill и одним workflow.

### Критерий понимания
Ты можешь довести маленькую задачу до результата и проверить её.

### Типичный провал
Брать слишком большой проект первым кейсом.

## Шаг 6. Научиться делать self-check и review
### Что сделать
Отдельно проверять:
- результат
- риски
- утечки
- слабые места

### Домашка
После любой задачи оформлять короткий self-review.

### Критерий понимания
Ты не веришь агенту на слово.

### Типичный провал
Считать первый ответ уже финальным.

## Шаг 7. Научиться строить свои skills и workflows
### Что сделать
Понять, когда тема достойна отдельного skill или workflow.

### Домашка
Сделать один свой skill из повторяющейся задачи и один workflow из повторяющейся процедуры.

### Критерий понимания
Ты не просто копируешь шаблон, а умеешь адаптировать систему под себя.

### Типичный провал
Создавать десятки навыков без реальной необходимости.
