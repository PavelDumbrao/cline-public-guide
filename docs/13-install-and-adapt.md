# 13. Install and Adapt

## Зачем этот файл
Этот файл отвечает на практический вопрос:
**как перенести этот repo в свой setup без тупняка и без слепого копирования чужой системы.**

## 1. Что копировать первым делом
Сначала копируй только:
- `ai/rules/`
- 3–5 нужных `ai/skills/`
- при необходимости `ai/hooks/TaskStart` и `ai/hooks/PreToolUse`
- 2–3 MCP из `docs/07-mcp-guide-human.md`

## 2. Что не копировать буквально
Нельзя копировать буквально:
- placeholders как реальные значения
- bot-specific настройки
- любые команды, если не понимаешь зачем они
- infra-specific части под чужой сервер
- n8n JSON без понимания, как они работают

## 3. Куда класть rules
Обычно rules кладут туда, где их будет читать твой Cline setup.

**Важно:** проверь свою версию Cline.
Пути и способ подключения могут отличаться.

Если у тебя уже есть текущая система rules:
- не смешивай сразу всё подряд
- добавляй по 1–2 правила и проверяй поведение

## 4. Куда класть skills
Обычно skills лежат в директории skills, где Cline их видит как отдельные папки с `SKILL.md`.

**Это шаблон, адаптируй под свой setup.**

Правило:
- 1 папка = 1 skill
- внутри должен быть `SKILL.md`

## 5. Куда класть hooks
Hooks клади только туда, где их реально читает твой Cline.

**Проверь свою версию Cline.**

На старте лучше использовать только:
- `TaskStart`
- `PreToolUse`

## 6. Как работать с workflows
Здесь есть **Cline markdown workflows** в `ai/workflows/`.
Они нужны как reusable procedures.

И отдельно есть **n8n JSON** в `examples/n8n/`.
Это не одно и то же.

## 7. Что заменить вручную
Обязательно вручную заменить:
- `<API_KEY>`
- `<BOT_TOKEN>`
- `<CHAT_ID>`
- `<PRIVATE_URL>`
- `<PRIVATE_IP>`
- любые env placeholders
- project-specific названия

## 8. Как проверить, что Cline это видит
Проверяй по шагам:
- агент придерживается правил языка/стиля
- нужный skill можно явно вызвать
- hook реально срабатывает на событии
- MCP реально используется по назначению

## 9. Типичные ошибки при переносе
- копировать всё сразу
- не понимать разницу между rule и skill
- ставить hooks до базового smoke test
- тянуть private-only MCP в обычный setup
- путать Cline workflows и n8n JSON

## 10. Минимальный smoke test после переноса
1. Проверь rule по языку и стилю
2. Проверь docs search
3. Проверь вызов одного skill
4. Проверь безопасное поведение команды
5. Если есть playwright — проверь простое открытие страницы
