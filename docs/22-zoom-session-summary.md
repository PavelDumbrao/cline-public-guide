## 🎯 Основные инсайты

- Cline даёт максимальную свободу кастомизации по сравнению с аналогами (CloudCode, KiloCode): можно подключать любые API и MCP-серверы.
- Китайские модели (на базе Claude Opus) могут галлюцинировать, если в soul указано, что они Cline. Практический вывод — временно отключать soul или переключать модель.
- Hooks жизненно важны для предотвращения зацикливания ИИ. Пример — автосброс микрозадачи, если она висит дольше 60 секунд.
- Семантический поиск кода (MCP Code Search) резко снижает расход токенов, потому что поиск идёт по смыслу, а не через чтение десятков файлов подряд.
- Сильная связка: ChatGPT / Perplexity как «архитектор», Cline как «исполнитель». Обмен можно строить через GitHub, чтобы не терять контекст и экономить дорогие токены.

## 💡 Практические выводы и решения

- Готовые Rules, Skills и структура папок вынесены в открытый GitHub-репозиторий для лёгкой адаптации под себя.
- Правила лучше держать на английском, если цель — экономить токены и упростить мышление LLM на базовом уровне.
- В качестве дешёвой рабочей модели можно использовать GPT-5.4 или MIMO V2 Pro, а Claude 4.6 Opus — оставлять для проектирования и финального рефакторинга.
- Для обхода ограничений Cloudflare при поиске можно использовать Playwright, чтобы Cline ходил в сеть через локальный браузер.
