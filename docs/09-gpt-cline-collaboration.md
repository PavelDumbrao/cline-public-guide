# 09. GPT + Cline Collaboration

## Практичный паттерн

### GPT / Perplexity / другой LLM
Используй как:
- архитектора
- генератор альтернатив
- исследователя
- критика

### Cline
Используй как:
- исполнителя
- редактора кода
- оператора терминала
- локального проверяющего

## Рабочая схема
1. Сначала просишь GPT предложить 2–3 варианта решения
2. Выбираешь лучший
3. Даёшь Cline чёткую задачу на реализацию
4. Потом делаешь отдельный цикл проверки

## Почему так лучше
Потому что один и тот же агент часто хуже одновременно:
- проектирует
- реализует
- сам себя проверяет

Разделение ролей уменьшает самообман.

## Не верь агенту на слово
Отдельно проверяй:
- действительно ли файл создан
- действительно ли тест проходит
- действительно ли endpoint отвечает
- действительно ли UI работает

## Хороший prompt для Cline
- короткий
- конкретный
- с критериями успеха
- с ограничениями
- без лишней абстракции
